# Footprints — Product Planning (from Figma Make)

## 1) Intended Function of the App

Footprints is a mobile-first wellbeing and social-impact app that nudges users to complete meaningful real-world activities ("prompts"), document completion, and share progress with friends.

At a product level, the app combines:

- **Personal habit activation** (prompt list + next action)
- **Guided education** (why activity matters + how to start)
- **Proof of completion** (photo + note + date + location)
- **Social reinforcement** (community feed)
- **Long-term reflection** (history + impact stats)

## 3) Workflow Summary (Current UX)

## 3.1 Entry + Auth

1. User opens app on `Login` screen.
2. User submits username/password.
3. App navigates directly to `/` (currently mocked, no real auth).

## 3.2 Primary Navigation

Bottom tab nav from `Root`:

- **Feed** (`/feed`)
- **Home** (`/`)
- **History** (`/history`)
- **Settings** (`/settings`)

## 3.3 Prompt Completion Journey

1. On `Home`, user sees active prompts + "time since last activity".
2. User taps a prompt card -> opens `/prompt/:id`.
3. Inside prompt, user switches between two subviews:
   - **Info** (`view=detail`) for context and instructions
   - **Upload** (`view=upload`) to submit evidence and details
4. User submits completion -> navigates back to home.

## 3.4 Social + Progress Journey

- `Feed`: view community/friend activity posts.
- `History`: review completed actions and aggregate impact stats.
- `Settings`: account/profile/preferences/help/logout scaffolding.

## 4) Product Goals

## 4.1 User Goals

- Find meaningful activities quickly.
- Understand why each activity matters.
- Complete and record activities with low friction.
- Track personal consistency over time.
- Feel motivated through social visibility and shared progress.

## 4.2 Business/Program Goals

- Increase weekly active participation in prompts.
- Improve completion rate per prompt.
- Build retention through streak/progress loops.
- Encourage community participation and positive social proof.

## 5) Core User Stories

## 5.1 Onboarding & Access

- As a new user, I want to sign in quickly so I can start activities.
- As a returning user, I want to land directly on my next best prompt.

## 5.2 Prompt Discovery & Action

- As a user, I want a clear list of active prompts so I know what to do next.
- As a user, I want to open a prompt and see why it matters so I stay motivated.
- As a user, I want actionable starter steps so I can begin immediately.

## 5.3 Completion & Evidence

- As a user, I want to upload a photo and short note so I can prove completion.
- As a user, I want to add date/location context so my activity feels meaningful and trackable.
- As a user, I want to choose whether to share with friends so I control visibility.

## 5.4 Reflection & Motivation

- As a user, I want to view my history so I can see consistency over time.
- As a user, I want aggregate stats so I can feel progress and impact.

## 5.5 Community

- As a user, I want to see friends’ activities so I feel inspired and connected.
- As a user, I want simple friend management so my feed stays relevant.

## 6) MVP Scope Recommendation

## 6.1 In MVP

- Mobile-first shell with bottom tabs.
- Prompt list (home) with prompt detail + upload flow.
- Basic completion creation (photo metadata, note, date, location).
- History list + basic stats.
- Read-only feed from seeded/mock or backend data.
- Minimal account settings page.

## 6.2 Out of MVP (Phase 2+)

- Full friend graph management.
- Rich moderation/reporting tools.
- Advanced recommendations/personalization.
- Desktop admin/dashboard experience.

## 7) Functional Requirements by Screen

## 7.1 Login

- Accept credentials.
- Handle success/failure states.
- Route to authenticated app shell.

## 7.2 Home

- Display active prompts.
- Show recency metric (time since last activity).
- Support navigation to prompt details.

## 7.3 Prompt Detail

- Show activity objective, rationale, starter steps, and resources.

## 7.4 Prompt Upload

- Capture image upload.
- Capture text description.
- Capture date + location.
- Toggle visibility/share option.
- Submit/cancel flows.

## 7.5 Feed

- List friend/community posts.
- Show core metadata: user, date, location, description, related activity.

## 7.6 History

- Show completed activities timeline/cards.
- Show impact summary stats.

## 7.7 Settings

- Show profile snapshot.
- Provide account/privacy/notification/help entry points.
- Provide logout action.

## 8) Data Model Draft (Planning)

Suggested entities:

- **User**: id, username, displayName, createdAt
- **Prompt**: id, title, description, category, guidance, active
- **PromptCompletion**: id, userId, promptId, photoUrl, note, date, location, visibility, createdAt
- **FeedPost**: id, completionId, userId, audience, createdAt
- **Friendship**: userId, friendId, status
- **UserStats**: userId, completedCount, streakDays, lastActivityAt

## 9) Risks and Gaps Identified

- Current Make code is largely mock-data driven (no production data wiring).
- Auth is placeholder navigation only.
- Upload UI is simulated (no real storage integration yet).
- Feed/friend management is conceptual; behavior rules need product decisions.
- Desktop dashboard artifacts may confuse scope unless explicitly deferred.

## 10) Delivery Plan (Phased)

## Phase 0 — Foundation

- Settle product scope: mobile app first, desktop deferred.
- Define backend choice (AWS + DynamoDB + S3).
- Define design tokens/component constraints for implementation.

## Phase 1 — MVP Core Flows

- Implement auth (real session flow).
- Implement prompt list/detail/upload with persistence.
- Implement history list + summary stats from real completions.
- Implement basic feed read path.

## Phase 2 — Social & Growth

- Friend management and visibility controls.
- Improve feed interactions (likes/comments optional).
- Add habit mechanics (streaks, reminders, prompt rotation).

## Phase 3 — Ops & Scale

- Moderation/reporting workflows.
- Analytics instrumentation and dashboards.
- Optional desktop/admin console (using dashboard template if desired).
